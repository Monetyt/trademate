## 一、技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | 单文件 HTML + Vanilla CSS/JS | 零构建，零依赖，GitHub Pages 直接托管 |
| AI | DeepSeek API（客户端直调） | 用户自带 Key，无服务端成本 |
| 存储 | localStorage | 产品信息 + 历史记录 + 主题偏好 |
| PWA | Service Worker | 离线缓存 + 可安装 + 推送 |
| 部署 | GitHub Pages | 免费，自动 HTTPS，全球 CDN |
| 图标 | Phosphor Icons（SVG sprite） | 开源 MIT，2000+ 图标，4KB gzip |

**不做的事**：不引入 React/Vue/构建工具——单文件 HTML 在当前规模下是最优解。

## 三、数据流

```
用户粘贴询盘
    │
    ▼
PromptEngine.build(邮件类型, 产品信息, 询盘内容, 输出语言)
    │
    ▼
APIClient.call(systemPrompt, userPrompt)
    │
    ▼  POST https://api.deepseek.com/v1/chat/completions
    │  Header: Authorization: Bearer {userKey}
    │  Body: { model: "deepseek-chat", messages, temperature: 0.3 }
    │
    ▼
响应 → 渲染到 ResultPanel → 存入 History (localStorage)
```

API Key 全程只在浏览器端使用，不经过任何第三方服务器。

## 六、关键性能指标

| 指标 | 目标 | 实现方式 |
|------|------|---------|
| FCP | < 1.5s | 单文件 HTML，无外部 CSS/JS 依赖 |
| LCP | < 2s | 系统字体栈，无 Web Font 加载 |
| TBT | < 100ms | 无重型 JS，事件委托 |
| API 响应 | < 8s | DeepSeek 平均 3-5s |
| 离线可用 | ✅ | Service Worker 缓存壳 + 最近 20 条历史 |
| PWA 安装 | ✅ | manifest.json + 注册 SW |

## 七、安全设计

- API Key → localStorage，永不上传
- 无服务端 → 无数据泄露风险
- 无第三方 CDN → 无供应链攻击面
- Content-Security-Policy 通过 GitHub Pages 默认配置
