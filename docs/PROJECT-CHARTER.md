# TradeMate 重构项目章程

## 项目目标

将 TradeMate 外贸商务邮件助手重构为具有 DeepSeek 级别设计品质的专业 SaaS 产品。

## 开发团队

| 角色 | 代号 | 职责 |
|------|------|------|
| 项目经理 (PM) | @pm | 节点把控、风险管理、跨角色协调 |
| 产品经理/需求 (PD) | @pd | 需求挖掘、用户故事、功能优先级 |
| UI/UX 设计师 | @ui | 视觉设计、交互规范、设计系统 |
| 系统架构师 | @sa | 技术选型、架构设计、性能指标 |
| 前端开发 | @fe | 页面实现、组件开发、交互效果 |
| 后端开发 | @be | API 设计、数据处理、服务部署 |
| 移动端开发 | @mobile | 响应式适配、PWA、移动体验 |
| 算法工程师 | @algo | Prompt 优化、输出质量控制 |
| 测试工程师 | @qa | 测试用例、Bug 追踪、验收报告 |
| 配置管理员 | @cm | 版本管理、部署流水线、环境配置 |

当前阶段：所有角色由 Claudian 统一扮演。后续可接入独立 Agent。

## 开发节点

| 节点 | 名称 | 交付物 | 评审方式 |
|------|------|--------|---------|
| G1 | 需求评审 | 产品需求文档 (PRD) | 用户确认 |
| G2 | UI/UX 评审 | 设计稿 + 交互规范 | 用户确认 |
| G3 | 技术方案评审 | 架构设计文档 | 用户确认 |
| G4 | 开发阶段 | 可运行代码 | 内部评审 |
| G5 | 测试验收 | 测试报告 | 用户确认 |
| G6 | 上线发布 | 部署 + 上线清单 | 用户确认 |
| G7 | 项目复盘 | 复盘报告 | 用户确认 |

每个节点完成后必须经用户确认才能进入下一节点。节点内部的问题由团队自行评审收敛。
